# yiliminav 部署指南

## 前置要求

- [ ] Cloudflare 账号
- [ ] MiniMax API Key（从 https://platform.minimax.io 获取）
- [ ] 域名 nav.361026.xyz（已购买并解析到Cloudflare）

---

## 步骤1：安装Wrangler CLI

```bash
npm install -g wrangler
wrangler login
```

---

## 步骤2：创建Cloudflare资源

### 2.1 创建KV命名空间

```bash
# 名言缓存
wrangler kv:namespace create QUOTE_KV
# 输出类似: kv_namespace_id = "xxxxx"

# 诗词缓存
wrangler kv:namespace create POEM_KV
```

### 2.2 创建R2存储桶（可选，用于AI封面）

```bash
wrangler r2 bucket create ai-covers
```

### 2.3 创建D1数据库（可选）

```bash
wrangler d1 create yiliminav-db
```

---

## 步骤3：配置wrangler.toml

将步骤2获取的ID填入 `wrangler.toml`：

```toml
[[kv_namespaces]]
binding = "QUOTE_KV"
id = "你的QUOTE_KV_ID"

[[kv_namespaces]]
binding = "POEM_KV"
id = "你的POEM_KV_ID"
```

---

## 步骤4：设置环境变量

```bash
# MiniMax API Key（必需）
wrangler secret put MINIMAX_API_KEY
# 输入你的MiniMax API Key
```

---

## 步骤5：部署

```bash
# 构建前端
npm run build

# 部署到Cloudflare Pages
wrangler pages deploy dist
```

---

## 步骤6：绑定域名

1. 登录 Cloudflare Dashboard
2. 进入 Pages 项目 → Custom Domains
3. 添加域名 `nav.361026.xyz`

---

## 快速命令汇总

```bash
# 登录
wrangler login

# 创建KV
wrangler kv:namespace create QUOTE_KV
wrangler kv:namespace create POEM_KV

# 设置密钥
wrangler secret put MINIMAX_API_KEY

# 部署
npm run build && wrangler pages deploy dist

# 本地预览
npm run dev:all
```

---

## 常见问题

### Q: 部署后显示空白？
A: 检查浏览器控制台是否有错误，确保API路由正确配置

### Q: AI功能不工作？
A: 确保已设置 MINIMAX_API_KEY 环境变量

### Q: 名言/诗词不显示？
A: 确保KV命名空间已正确配置

### Q: 如何更新部署？
```bash
npm run build
wrangler pages deploy dist --project-name=yiliminav
```
