## 四、Cloudflare配置

### 4.1 登录Wrangler

```bash
# 安装Wrangler
npm install -g wrangler

# 登录（会打开浏览器）
wrangler login
```

### 4.2 创建KV命名空间

```bash
# 名言缓存（TTL: 24小时）
wrangler kv:namespace create QUOTE_KV
# 记录输出的 id，例如: kv_namespace_id = "0a2e4c123456789"

# 诗词缓存（TTL: 1小时）
wrangler kv:namespace create POEM_KV
# 记录输出的 id
```

### 4.3 创建R2存储桶（可选）

```bash
# AI封面存储
wrangler r2 bucket create ai-covers
```

### 4.4 创建D1数据库（可选）

```bash
# 用户数据存储
wrangler d1 create yiliminav-db
```

## 五、配置wrangler.toml

### 5.1 编辑配置文件

打开 `wrangler.toml`，填入上一步获取的ID：

```toml
name = "yiliminav"
compatibility_date = "2024-01-01"
pages_build_output_dir = "dist"

[vars]
APP_NAME = "yiliminav"
APP_VERSION = "1.0.0"

# KV命名空间 - 填入你的ID
[[kv_namespaces]]
binding = "QUOTE_KV"
id = "0a2e4c123456789"  # 替换为你的ID

[[kv_namespaces]]
binding = "POEM_KV" 
id = "0a2e4c123456790"  # 替换为你的ID

# R2存储桶（可选）
[[r2_buckets]]
binding = "AI_COVERS"
bucket_name = "ai-covers"

# D1数据库（可选）
# [[d1_databases]]
# binding = "DB"
# database_name = "yiliminav-db"
# database_id = "xxx"
```

### 5.2 设置API密钥

```bash
# 必须：MiniMax API密钥
wrangler secret put MINIMAX_API_KEY
# 输入你的MiniMax API Key（从 https://platform.minimax.io 获取）

# 可选：数据库ID（如果使用D1）
# wrangler secret put D1_DATABASE_ID
```

## 七、绑定域名

### 7.1 通过Dashboard配置

1. 登录 [Cloudflare Dashboard](https://dash.cloudflare.com)
2. 进入 **Pages** → **yiliminav** → **Custom domains**
3. 点击 **Set up a custom domain**
4. 输入 `nav.361026.xyz`
5. 点击 **Continue** → **Activate Domain**

### 7.2 通过命令行

```bash
wrangler pages domain add nav.361026.xyz
```

## 十、CI/CD自动化部署（可选）

### 10.1 创建GitHub Actions

在仓库根目录创建 `.github/workflows/deploy.yml`：

```yaml
name: Deploy to Cloudflare Pages

on:
  push:
    branches:
      - main

jobs:
  deploy:
    runs-on: ubuntu-latest
    permissions:
      contents: read
      deployments: write

    steps:
      - uses: actions/checkout@v4

      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: 20

      - name: Install dependencies
        run: npm ci

      - name: Build
        run: npm run build
        env:
          MINIMAX_API_KEY: ${{ secrets.MINIMAX_API_KEY }}

      - name: Deploy to Cloudflare Pages
        uses: cloudflare/pages-action@v1
        with:
          apiToken: ${{ secrets.CLOUDFLARE_API_TOKEN }}
          accountId: ${{ secrets.CLOUDFLARE_ACCOUNT_ID }}
          projectName: yiliminav
          directory: dist
          gitHubToken: ${{ secrets.GITHUB_TOKEN }}
```

### 10.2 配置Secrets

在GitHub仓库 Settings → Secrets and variables → Actions 中添加：

| Secret Name | 获取方式 |
|-------------|----------|
| CLOUDFLARE_API_TOKEN | Cloudflare Profile → API Tokens → Create Token (使用 Edit Cloudflare Pages 模板) |
| CLOUDFLARE_ACCOUNT_ID | Cloudflare Dashboard URL 中的账户ID |
| MINIMAX_API_KEY | MiniMax平台获取 |
