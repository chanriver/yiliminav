## 四、数据库设计

### 4.1 D1 表结构

```sql
-- 用户配置表
CREATE TABLE IF NOT EXISTS users (
  id TEXT PRIMARY KEY,
  email TEXT UNIQUE,
  name TEXT,
  created_at INTEGER DEFAULT (unixepoch()),
  updated_at INTEGER DEFAULT (unixepoch())
);

-- 书签分类表
CREATE TABLE IF NOT EXISTS categories (
  id TEXT PRIMARY KEY,
  user_id TEXT NOT NULL,
  name TEXT NOT NULL,
  icon TEXT,
  color TEXT,
  sort_order INTEGER DEFAULT 0,
  created_at INTEGER DEFAULT (unixepoch()),
  updated_at INTEGER DEFAULT (unixepoch()),
  FOREIGN KEY (user_id) REFERENCES users(id)
);

-- 书签链接表
CREATE TABLE IF NOT EXISTS links (
  id TEXT PRIMARY KEY,
  user_id TEXT NOT NULL,
  category_id TEXT,
  title TEXT NOT NULL,
  url TEXT NOT NULL,
  icon TEXT,
  cover_url TEXT,
  description TEXT,
  sort_order INTEGER DEFAULT 0,
  created_at INTEGER DEFAULT (unixepoch()),
  updated_at INTEGER DEFAULT (unixepoch()),
  FOREIGN KEY (user_id) REFERENCES users(id),
  FOREIGN KEY (category_id) REFERENCES categories(id)
);

-- 用户配置表
CREATE TABLE IF NOT EXISTS settings (
  id TEXT PRIMARY KEY,
  user_id TEXT NOT NULL,
  key TEXT NOT NULL,
  value TEXT,
  FOREIGN KEY (user_id) REFERENCES users(id)
);
```

### 4.2 KV 缓存策略

| Key | 值类型 | TTL | 说明 |
|-----|--------|-----|------|
| `daily_quote` | JSON | 86400 | 每日名言 |
| `poem_cache` | JSON | 3600 | 当前诗词 |
| `user:{id}:config` | JSON | 3600 | 用户配置缓存 |

## 五、MiniMax API 集成

### 5.1 API 端点

```
POST https://api.minimax.chat/v1/text/chatcompletion_v2
```

### 5.2 请求格式

```typescript
interface MiniMaxRequest {
  model: 'MiniMax-M2.5';
  messages: Array<{
    role: 'system' | 'user' | 'assistant';
    content: string;
  }>;
  stream?: boolean;
  temperature?: number;
  max_tokens?: number;
}
```

### 5.3 Prompt 模板

#### 推荐网站
```prompt
你是一个网站导航助手。用户正在寻找: {query}
可用分类: {categories}
请根据用户需求，从分类中选择最合适的，或者返回"custom"表示需要自定义分类。
返回JSON格式: {"category": "分类名", "confidence": 0.95}
```

#### 自然语言搜索
```prompt
用户搜索词: {query}
书签数据: {bookmarks}
请找出最匹配的书签，返回前5个。
返回JSON格式: [{"id": "xxx", "title": "xxx", "url": "xxx", "score": 0.9}]
```

#### AI归类
```prompt
网址: {url}
标题: {title}
可用分类: {categories}
请判断这个链接应该归到哪个分类。
返回JSON格式: {"category": "分类名", "reason": "原因"}
```

## 八、性能优化方案

### 8.1 静态资源

| 优化项 | 实现方式 | 预期效果 |
|--------|----------|----------|
| 资源压缩 | Vite production build |减小体积 70% |
| 分级背景 | blur→image→video | 首屏 < 1s |
| 图片WebP | Vite 图片转换 | 体积减小 30% |
| CDN 缓存 | Cloudflare 边缘 | TTFB < 100ms |

### 8.2 数据请求

| 优化项 | 实现方式 | 预期效果 |
|--------|----------|----------|
| SSR 注入 | 后端直出名言 | FCP < 500ms |
| SWR 缓存 | 先缓存后请求 | 加载 < 100ms |
| 请求合并 | 单次获取多数据 | 减少 RTT |

### 8.3 代码分割

| 优化项 | 实现方式 |
|--------|----------|
| 路由分割 | React.lazy() |
| 组件懒加载 | 装饰性功能延迟 |
| Worker | 倒计时独立线程 |
